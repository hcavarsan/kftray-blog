kftray includes a built-in MCP server that lets external clients browse clusters, manage port-forwards, and handle saved configurations through kftray.

For the full tool list and usage examples, see the [MCP interface page](/docs/interfaces/mcp).

## Enabling

Open Settings in kftray, find the MCP Server section, and toggle it on. Choose a port (default 3000). The server starts immediately and auto-starts on future launches while the toggle is enabled.

| Setting | Default | Description |
|---------|---------|-------------|
| Enabled | `false` | Whether the MCP server starts with the app |
| Port | `3000` | Port the server listens on |

## Connecting a client

The server endpoint is:

```
http://localhost:<port>/mcp
```

Clients that read server definitions from a config file typically need:

```json
{
  "mcpServers": {
    "kftray": {
      "url": "http://localhost:3000/mcp"
    }
  }
}
```

Where this file lives depends on the client. Check its documentation.

## Health check

```bash
curl http://localhost:3000/health
```

```json
{
  "status": "healthy",
  "server": "kftray-mcp",
  "protocol_version": "2024-11-05"
}
```

<Callout type="warn" title="Local only by default">
The server binds to `127.0.0.1`. Only processes on the same machine can reach it.
</Callout>
