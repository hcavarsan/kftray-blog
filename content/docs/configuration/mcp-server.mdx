title: MCP server

kftray includes a built-in MCP (Model Context Protocol) server that exposes port-forwarding operations over HTTP. MCP-compatible clients can call these operations as tools using JSON-RPC 2.0.

For the full tool reference, see the [MCP interface page](/docs/interfaces/mcp).

## Enabling

Open Settings in kftray, find the MCP Server section, and toggle it on. Choose a port (default 3000). The server starts immediately and auto-starts on future launches while the toggle is enabled.

| Setting | Default | Description |
|---------|---------|-------------|
| Enabled | `false` | Whether the MCP server starts with the app |
| Port | `3000` | HTTP port for the MCP server |

## Connecting a client

The server endpoint is:

```
http://localhost:<port>/mcp
```

For clients that read MCP server definitions from a config file:

```json
{
  "mcpServers": {
    "kftray": {
      "url": "http://localhost:3000/mcp"
    }
  }
}
```

Refer to the MCP client's own documentation for where to place this.

## Health check

```bash
curl http://localhost:3000/health
```

```json
{
  "status": "healthy",
  "server": "kftray-mcp",
  "version": "0.27.28",
  "protocol_version": "2024-11-05"
}
```

## What the server exposes

16 tools in three categories:

- **Kubernetes discovery** -- list contexts, namespaces, services, pods, and ports
- **Port-forward management** -- start, stop, and list active forwards
- **Configuration management** -- create, read, update, delete, export, and import configs

All operations use the same underlying engine as the desktop and terminal interfaces. Configs created through the MCP server appear in the GUI and TUI, and the other way around.

<Callout type="warn" title="Local access only">
The server binds to `127.0.0.1` by default. It is only accessible from the local machine.
</Callout>
