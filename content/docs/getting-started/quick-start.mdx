# Quick Start

In this guide, you'll create a port-forward configuration, start it, and verify that traffic flows from your local machine to a Kubernetes service.

## Prerequisites

- kftray or kftui [installed](/docs/getting-started/installation)
- A running Kubernetes cluster accessible via `kubeconfig`
- At least one service running in the cluster (we'll use a web service on port 80 as an example)

Verify your cluster access first:

```bash
kubectl get services -n default
```

## Step 1 — Create a Configuration

<Tabs items={['Desktop', 'Terminal']}>
<Tab value="Desktop">

<Steps>
<Step>
Click the kftray system tray icon to open the main window.
</Step>
<Step>
Click the **+** button (or press `Ctrl+N` / `Cmd+N`).
</Step>
<Step>
Fill in the configuration form:

| Field | Value |
|---|---|
| Service | `my-service` |
| Namespace | `default` |
| Local Port | `8080` |
| Remote Port | `80` |
| Protocol | `tcp` |
| Alias | `web` |
</Step>
<Step>
Click **Save**.
</Step>
</Steps>

</Tab>
<Tab value="Terminal">

Create a JSON configuration file:

```json
[
  {
    "service": "my-service",
    "namespace": "default",
    "local_port": 8080,
    "remote_port": 80,
    "context": "my-cluster",
    "workload_type": "service",
    "protocol": "tcp",
    "alias": "web"
  }
]
```

Save it as `config.json`, then import it in kftui:

<Steps>
<Step>
Launch `kftui`.
</Step>
<Step>
Press `i` to open the import dialog.
</Step>
<Step>
Navigate to your `config.json` file and press `Enter`.
</Step>
</Steps>

</Tab>
</Tabs>

## Step 2 — Start the Port Forward

<Tabs items={['Desktop', 'Terminal']}>
<Tab value="Desktop">

Toggle the switch next to your "web" configuration, or click **Start All** to enable all forwards.

A green status indicator means the connection is active.

</Tab>
<Tab value="Terminal">

<Steps>
<Step>
Use arrow keys to highlight the "web" configuration.
</Step>
<Step>
Press `Space` to select it.
</Step>
<Step>
Press `f` to start forwarding.
</Step>
</Steps>

The status column changes to **Running**.

</Tab>
</Tabs>

## Step 3 — Verify the Connection

Test that traffic flows through the forward:

```bash
curl http://localhost:8080
```

You should see the response from your Kubernetes service. For database services, connect your client to `localhost:LOCAL_PORT` as usual.

## Step 4 — Test Automatic Reconnection

Restart the target pod to see automatic reconnection in action:

```bash
kubectl rollout restart deployment/my-service -n default
```

Watch the kftray status — it briefly shows a reconnection attempt, then returns to green/running. No manual intervention needed.
